# 🔍 KubeLens

KubeLens is an AI‑powered Kubernetes log analysis API that runs on ARM64.  
It leverages a Retrieval‑Augmented Generation (RAG) engine to identify cluster anomalies and suggest remediations based on an embedded knowledge base.

## 🚀 Key Features

- **ARM64 native** – optimized for ARM‑based infrastructure (e.g., Scaleway Kapsule nodes).
- **Async API** – fully asynchronous FastAPI endpoints using `httpx` for non‑blocking I/O.
- **Dependency Injection** – settings are injected per request via FastAPI `Depends`.
- **RAG Engine** – vector indexing via Qdrant and LLM analysis via Ollama.
- **Secure by design** – integrated with Kubernetes External Secrets (ESO) for credential management and Zot OCI Registry for secure image storage.
- **Modern tooling** – PEP 621 `pyproject.toml` with exact version pins, `ruff` formatting, `mypy` typing, and `deptry` dependency checks.
- **CI/CD** – GitLab CI pipeline runs lint, tests, security scans and builds an ARM64 OCI image.

## 🏗️ Technical Stack

- **Backend:** FastAPI v0.135.3, Pydantic v2.12.5, `uvicorn` 0.43.0
- **Vector DB:** Qdrant client 1.17.1
- **LLM integration:** Ollama (`nomic-embed-text`, `qwen2.5:0.5b`)
- **HTTP client:** httpx 0.28.1
- **CI tools:** ruff, mypy, deptry, pytest, trivy
- **Container runtime:** Docker + Buildah (arm64)

## 🛠️ Quick Start

### Local Development

```bash
# Install in editable mode with dev tools
pip install -e ".[dev]"

# Run the server
python -m app.server
```

The API is available at `http://localhost:8000`.  
Health‑check: `GET /health` → `{"status":"OK"}`.  
Analyze endpoint: `POST /analyze` with JSON body `{"content":"<log>"}` returns a NDJSON stream.

### Build for Production (ARM64)

```bash
# Build using Buildah on an ARM‑compatible runner
buildah build \
  --arch arm64 \
  -t oci.vvtechsolutions.eu/kube-lens:latest .
```

The image can be pushed to your registry and deployed on any ARM64 Kubernetes cluster.

## 📄 Documentation

ReDoc: `http://localhost:8000/redoc`

## 🧪 Testing

Run tests with:

```bash
pytest --cov=src --cov-report=xml
```

The test suite covers the AI service logic and API endpoints.

## 📦 CI/CD

The `.gitlab-ci.yml` pipeline performs:

1. **Linting** – `ruff` and `mypy`.
2. **Dependency checks** – `deptry`.
3. **Unit tests** – `pytest`.
4. **Container build** – Buildah cross‑compile for ARM64.
5. **Security scan** – Trivy on the built image.
